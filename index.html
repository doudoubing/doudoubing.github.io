js兼容性问题

JS的兼容性问题（总结）:

1.注册事件的兼容处理
 if(obj.addEventListener){
    obj.addEventListener(type,handler,false);
  }else{
    obj.attachEvent('on'+type,handler);
  }
  2.解除绑定的事件兼容处理
  if(obj.removeEventListener){
    obj.removeEventListener(type,handler);
  }else{
    obj.detachEvent('on'+type,handler);
  }
3.阻止冒泡的兼容处理
  if(event.stopPropagation)
   {
    //标准浏览器
     event.stopPropagation();
   }else{
    //ie6,7,8
     event.cancleBubble = true;
   }
4.阻止默认行为的兼容处理
  if(e.preventDefault){
    //标准浏览器
    e.preventDefault();
   }else{
    //IE678
    e.returnValue = false;
   }
5.事件对象的兼容处理
  e = e || window.event;
6.获取触发事件的对象
  var tar = e.target || e.srcElement;

hexo部署到gitbub

将hexo部署到github上的几个步骤：

1.下载hexo及初始化

  • npm insatll hexo-cli -g
  • hexo init blog
  • cd blog
  • npm insatll
  • hexo server

2.在部署之前更改_config.yml文件改为

deploy:
    type: github
    repository: git@github.com:zhchnchn/zhchnchn.github.io.git
    branch: master

3.当遇到部署提示找不到Git

npm install hexo-deployer-git --save

4.部署三部曲

hexo clean 
hexo generate (hexo g)
hexo deploy (hexo d)

5.常用命令

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help  # 查看帮助
hexo version  #查看Hexo的版本

6.hexo配置

1.主题及标题和作者配置
    1）找到根目录下的_config.yml，将theme:你想要的主题名
    2）找到根目录下的_config.ym，将author:你的名字
    3）找到根目录下的_config.ym，将title:你的标题名
2.头像
    找到你应用的主题的_config.yml配置文件，将avar:头像地址
