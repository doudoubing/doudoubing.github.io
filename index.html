js兼容性问题

JS的兼容性问题（总结）:

1.注册事件的兼容处理
 if(obj.addEventListener){
    obj.addEventListener(type,handler,false);
  }else{
    obj.attachEvent('on'+type,handler);
  }
  2.解除绑定的事件兼容处理
  if(obj.removeEventListener){
    obj.removeEventListener(type,handler);
  }else{
    obj.detachEvent('on'+type,handler);
  }
3.阻止冒泡的兼容处理
  if(event.stopPropagation)
   {
    //标准浏览器
     event.stopPropagation();
   }else{
    //ie6,7,8
     event.cancleBubble = true;
   }
4.阻止默认行为的兼容处理
  if(e.preventDefault){
    //标准浏览器
    e.preventDefault();
   }else{
    //IE678
    e.returnValue = false;
   }
5.事件对象的兼容处理
  e = e || window.event;
6.获取触发事件的对象
  var tar = e.target || e.srcElement;

angular入门01

Angular的核心特性

  • MVC
  • 模块化
  • 自动化双向数据绑定
  • 指令系统

下载angular(两种方式任选其一)

  • bower install angular –save
  • npm install angular –save

重要的指令

  • ng-app

    • 所有需要ng管理的代码必须被包裹在一个有ng-app指令的元素中
    • ng-app是ng的入口，表示当前元素的所有指令都会被angular管理（对每一个指令进行分析和操作）
    • 当angular执行ng-app的代码时会开辟一块空间，这块空间用来存储页面上可能会用到的数据模型

  • ng-model是双向数据绑定的指令

Angular特点

  • 最大程度的减少页面的DOM操作

示例

<body>
      <div ng-app>
    <h2>使用NG实现双向数据绑定</h2>
    <input type="text" ng-model="user.name" placeholder="请输入用户名">
    <p>hello <strong>{{user.name}}</strong></p>
      </div>
 <script src="../node_modules/angular/angular.min.js"></script>
</body>

面向对象02

原型链

  • 什么是原型链？
    • 原型链是由对象组成的有限对象链，用于实现继承和共享属性。
  • 原型链的作用？
    • 实现数据共存、继承
  • 原型链的本质？
    • 实例对象与原型形成的链，链之间的联系通过proto。
  • 没有办法通过实例修改原型上的内容。
  • 当要找的某个属性找到了，就不会再向上找了，如果没找到就会按着原型链继续往上找直到找到。

继承的思想：别人给了我，我的还是我的

- 构造函数
- 函数也是对象（是函数也是对象 jquery里内部体现了这两种角色
 $('div') $是函数    $.ajax(); $是对象）

面向对象01

什么是面向对象？

  • 面向对象是一种编程模式

    • 好处：1.可维护性高 2.重用性高

  • 构造函数、实例对象、原型三者的关系？

    • 所有的构造函数都有一个原型属性prototype,属性值也是一个对象（new Object()）原型对象
    • 实例对象都有一个默认的属性__proto__，该属性不是标准（每个浏览器的这个属性的名
      称不一样）在实际开发中不可以使用这个属性，该属性是浏览器自己使用，该属性的指向和原
      型prototype指向相同。
      • 3.原型对象（prototype）里面都有一个constructor属性，该属性指向原型所属的构造函数。

  • 原型

    • 实现数据共享
    • 构造函数中原型属性prototype指向的区域被所有的实例共享
    • prototype是从构造函数的角度来说的。
    • __proto__是从实例对象的角度来说的
  • this
    • 构造函数中的this
    • 原型方法中的this
    • 事件函数中的this

  • new 实际上背后做了什么？

    • 分配内存，用来存储具体的数据。
    • 分配好之后用this指向这块区域（this存储内存地址）实际上this就是一个内存地址
    • 接下来会通过this指向的这块区域添加属性，并且存放数据。
    • 返回this(内存地址)

总结

一般情况下，方法放在原型上，属性放到构造函数中，达成的效果是：所有实例共享方法，但数据各自有一份。

HTML501

认识HTML5

  • HTML5是WHATWG和W3C博弈后的结果，它的牵头人是WHATWG.
  • HTML5简称H5.
  • 狭义HTML5:html5新的语义化标签，表单元素等。
  • 广义HTML5：HTML5+CSS3+JS
  • HTML5不仅仅是html的最新的一个版本，而且它制定了一系列的应用开发标准，是将web应用到应用开发平台的第一个HTML语言。
  • HTML5引进了一些新的元素，例如：语义化标签，智能表单等,方便我们创建富互联网应用制定了一系列的javascriptAPI，例如：地理定位，重力感应等。还可以结合canvas开发网络游戏。

语法规范

  1. 更加宽松
  2. 更加简洁
  3. 在实际开发中，应规范书写，不要太随意。

什么是语义化标签

  • 赋予标签意义，用合适的标签描述web结构。

  • 常用的语义化标签

    • header 头部标签
    • nav 导航标签
    • section 区域标签 （sect + ion 分+不同）
    • article 文章标签
    • aside 侧边栏标签
    • footer 页脚标签

  • section与div的区别

    • section比div更具有语义化

  • 使用新的语义化标签的好处

    • 对搜索引擎（百度，goole）友好。
    • SEO(search engine optimizer)搜索引擎优化
    • 有了良好的结构和语义的网页内容更容易被搜索引擎抓取，且获取到更高的权重。在用户搜索相关内容时，排名会靠前，自然会获得更高的流量。
    • 新的语义化标签从IE9开始兼容。

表单(带缓存，如果按F5是无法刷新表单的，按ctrl+F5用来强制刷新表单，或者重新打开页面)

  • 随着互联网应用和移动开发的发展，传统的表单越来越不能满足我们的需要了。HTML5对表单进行了调整
  • 新的表单元素类型
    • 邮箱类型：type=”email”
    • 拾色器： type=”color”
    • 网址： type=”url”
    • 数字： type=”number”（值可以输入数字，可以输入e科学计数法）
    • 范围： type=”range”
    • 搜索框： type=”search”（与text暂时是一样的）
    • 电话号码： type=”tel”
    • 日期： type=”date”(有日期组件)
    • 时间： type=”time”
    • 日期和时间：type=”datetime”(通常浏览器不支持)
    • 周数： type=”week”
    • 月： type=”month”
    • 本地日期和时间：type=”datetime-local”
    • <fieldset></fieldset>表单框
    • <legend></legend>表单标题
  • 新增的表单元素
    • datalist 数据列表
    • datalist是隐藏在表单域中的，不显示的，使用它里面数据的表单元素应该用list属性绑定datalist的id
    • output 输出
    • <output></output> 用于输出计算结果的

  • 新的表单属性

    • 自动获取焦点： autofocus（常用）
    • 必填项： required（常用）
    • 自动完成： autocomplete（常用）可写在form上（使表单域下的所有表单元素都会自动完成）
    • 禁止自动完成： autocomplete=”off”
    • 占位符： placeholder=”请输入用户名”
    • 正则表达式： pattern=”正则表达式” []表示选择的范围 {}表示选择几个
    • 多个邮箱： multiple(允许输入多个邮箱或file)逗号分割
    • 多个文件： multiple
      -表单域外的一个元素：form=”表单域的id”在表单提交的时候将它也一起提交，表示该表单元素属于这个表单域。

  • 新的表单事件

    • oninvalid 表单元素验证不通过时，触发

    • oninput 当用户输入时，触发

    • oninput与onchange的区别

      1.oninput在用户一输入时就触发，onchange是用户输入后，并且该表单元素失去焦点时才触发。
      2.oninput只能作用于input和textarea;onchange可以作用于input，textarea和select等
      3.oninput的兼容性没有onchange好。

hexo部署到gitbub

将hexo部署到github上的几个步骤：

1.下载hexo及初始化

  • npm insatll hexo-cli -g
  • hexo init blog
  • cd blog
  • npm insatll
  • hexo server

2.在部署之前更改_config.yml文件改为

deploy:
    type: github
    repository: git@github.com:zhchnchn/zhchnchn.github.io.git
    branch: master

3.当遇到部署提示找不到Git

npm install hexo-deployer-git --save

4.部署三部曲

hexo clean 
hexo generate (hexo g)
hexo deploy (hexo d)

5.常用命令

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help  # 查看帮助
hexo version  #查看Hexo的版本

6.hexo配置

1.主题及标题和作者配置
    1）找到根目录下的_config.yml，将theme:你想要的主题名
    2）找到根目录下的_config.ym，将author:你的名字
    3）找到根目录下的_config.ym，将title:你的标题名
2.头像
    找到你应用的主题的_config.yml配置文件，将avar:头像地址
